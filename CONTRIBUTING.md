## How to contribute

1. Fork the repository.
2. Create a branch for your change.
3. Open a pull request describing what you changed and why.
4. The maintainer will review, request changes if needed, and merge at their discretion. Not all PRs will be accepted.

## What's welcome

Bug fixes, performance improvements, accessibility fixes, and small well-scoped features are generally the easiest to review and merge. For larger changes, consider opening an issue first to discuss the approach.
